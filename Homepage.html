Design and Implementation

Objectives

The objectives of this chapter are to introduce object-oriented software design using the UML and highlight important implementation concerns. When you have read this chapter, you will:

  • Understand the most important activities in a general, objectoriented design process;
  • Understand some of the different models that may be used to document an object-oriented design;
  • Know about the idea of design patterns and how these are a way reusing design knowledge and experience;
  • Have been introduced to key issues that have to be considered when implementing software, including software reuse and open-source development.

7.1 Object-oriented design using the UML

Software design and implementation involves developing a working system by planning its structure and components using object-oriented design to ensure it operates effectively, integrates smoothly with other systems, and remains easy to develop, update, and manage.

7.2 Design patterns

Design patterns are reusable ideas that solve common software design problems by providing proven solutions and examples, helping developers build systems that are flexible, efficient, and easy to understand and maintain.

7.3 Implementation issues

Software implementation involves creating an executable system through reuse of existing software, configuration management to control changes, and host-target development to ensure the software runs correctly on its intended platform.

7.4 Open source development

Open source development is a software approach where source code is made publicly available for anyone to use, modify, and improve, encouraging collaboration and faster innovation while raising licensing, control, and business model considerations.
